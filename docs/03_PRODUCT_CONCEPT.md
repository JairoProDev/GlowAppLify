# Product Concept: The Execution Board

## 1. From Vision Board to Execution Board

### 1.1 The Evolution

**Vision Board** (Mediocre):
- Collage of aspirational images
- Decorative, passive
- No action plan
- No tracking
- Activates dopamine prematurely â†’ Reduces urgency

**Execution Board** (Elite):
- Hierarchical system of data + purpose
- Strategic, active
- Daily micro-actions
- Real-time metrics
- Activates prefrontal cortex â†’ Drives execution

### 1.2 The Three States of Psychology

An Execution Board couples three brain systems:

1. **Desire** (Limbic System):
   - Visual anchors (Future Self images)
   - Emotional connection to goal
   - *Why* you want this

2. **Strategy** (Prefrontal Cortex):
   - SMART goals
   - KPIs and deadlines
   - *What* you're achieving

3. **Habit** (Basal Ganglia):
   - Daily micro-actions
   - Implementation Intentions
   - *How* you execute

**Result**: Inspiration â†’ Organization â†’ Automation

## 4. The 4 Action Patterns (DNA of Goals)

Beyond *what* (vertical), goals have *how* (pattern):

## 5. User Journey: From Onboarding to Execution

### 5.1 The Complete Flow

```
User Entry â†’ Onboarding (3-5 min) â†’ AI Generation (30 sec) â†’ Board Presentation â†’ Daily Execution â†’ Weekly Review
```

### 5.2 Onboarding: The 4-Step Interview

**STEP 1: The Big Goal**
- Question: "What's the ONE goal you want to achieve in 90 days?"
- User input: Free text or category selection
- AI processing: Extract goal_type, domain, timeframe

**STEP 2: Life Context**
- Question: "What's your typical day like?"
- Options: Student / 9-to-5 / Entrepreneur / Parent / Mixed
- AI processing: Determines available time blocks, friction points

**STEP 3: Obstacle Identification** (Mental Contrasting)
- Question: "What stopped you last time you tried this?"
- Options: Time / Don't know how / Lose motivation / Perfectionism / etc.
- AI processing: Generates if-then plans for each obstacle

**STEP 4: Future Self Vision**
- Question: "Imagine 90 days from now, you achieved this. How do you feel?"
- User input: 2-4 sentences
- AI processing: Extracts emotional anchors for vision layer

**Critical**: Each step allows "Skip" â†’ Generates generic board if user wants speed

### 5.3 AI Generation Process (Behind the Scenes)

**Input**: Onboarding responses

**Processing**:
1. Parse goal â†’ SMART structure (Specific, Measurable, Achievable, Relevant, Time-bound)
2. Break into weekly milestones
3. Generate 5-7 daily micro-actions
4. Create if-then plans for obstacles
5. Design morning/evening rituals based on time availability
6. Set KPIs (auto-generated from goal type)

**Output**: Complete Execution Board

**Time**: 30 seconds (with artificial minimum to build trust)

## 6. Key Product Principles

### 6.1 Simplification Over Flexibility

**Anti-Pattern**: Notion's infinite flexibility â†’ Analysis paralysis

**Bloom's Approach**:
- 1 active board (MVP)
- Max 5 actions/day
- Max 3 life verticals at once
- No customization until Week 2

**Why**: Paradox of choice â†’ Fewer options = Better outcomes

### 6.2 AI Does the Thinking, User Executes

**User Should NEVER**:
- Manually create SMART goals
- Plan daily actions
- Schedule routines
- Calculate KPIs

**User ONLY**:
- States vague goal
- Answers 4 onboarding questions
- Checks off completed actions

**AI Handles**:
- Goal structuring
- Action breakdown
- Obstacle anticipation
- Ritual design
- Progress calculation

### 6.3 Process Over Results

**Traditional Apps**: Celebrate outcome (lost 10kg)

**Bloom**: Celebrate consistency (10 days straight)

**Why**:
- Outcomes = Lagging indicator (takes months)
- Consistency = Leading indicator (visible daily)
- Gamification on process â†’ Sustainable
- Gamification on results â†’ Burnout

### 6.4 Evidence-Based Progress (Proof Layer)

**The Problem**: Self-reported progress = Prone to bias

**The Solution**: Proof layer

| Goal Type | Proof Method |
|-----------|-------------|
| Fitness | Integrations (Strava, Apple Health) |
| Learning | Quiz scores, certificates |
| Financial | Bank sync (Plaid), manual entry |
| Habit | Photo upload, check-in |

**Future**: Wearables (Oura, Whoop) for bio-metrics

## 8. The "ONE Thing" Philosophy

**Inspiration**: Gary Keller's book "The One Thing"

**Focusing Question**: 
"What's the ONE thing I can do today, such that by doing it, everything else becomes easier or unnecessary?"

**Bloom Application**:
- AI identifies highest-impact action each day
- Displayed prominently in daily view
- User encouraged to do this FIRST

**Psychological Benefit**:
- Eliminates decision fatigue
- Creates momentum (early win)
- Prevents "productive procrastination"

## 9. Gamification Design (Anti-Addiction)

### 9.1 What We Gamify

âœ… **Healthy Gamification**:
- Streaks (consistency)
- Completion percentage
- Obstacle overcome count
- "Executor Score" (adherence to system)

### 9.2 What We DON'T Gamify

âŒ **Unhealthy Gamification**:
- Infinite scroll
- Likes/hearts
- Leaderboards (public comparison)
- Points for logging in (dark pattern)

### 9.3 The Streak Design

**Traditional Streak** (e.g., Duolingo):
- Lose streak if you miss 1 day â†’ Anxiety
- "Streak freeze" purchasable â†’ Monetization manipulation

**Bloom's Streak**:
- Grace period: 1 missed day doesn't break streak
- Focus: "80% completion = success" (not 100%)
- No punishment, just neutral
- Milestone celebrations: 7, 21, 66 days

**Why**: We want consistency, not perfectionism.

## 10. The Identity Shift Framework

**Core Insight**: Outcome goals fail. Identity goals stick.

**Traditional Goal**: "I want to lose 10kg"
- Focus: External result
- Motivation: Willpower-dependent

**Bloom's Approach**: "I am becoming an athlete"
- Focus: Internal identity
- Motivation: Self-image alignment

**Onboarding Question**:
"Who do you need to BECOME to achieve this?"

**Daily Reminder**:
"Today, you acted as [identity]. Keep going."

**Psychological Mechanism**:
- Cognitive dissonance: "I'm an athlete" vs "I'm skipping workouts" â†’ Discomfort
- Resolution: Act like athlete to reduce dissonance
